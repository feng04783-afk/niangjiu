# 实机动力学参数辨识落地指南

本指南面向“把当前仓库里的离线轨迹优化与回归矩阵能力，落地到实机辨识”的工程实施。

## 1. 先明确目标

建议把目标拆成 3 层：

1. **可运行**：机械臂能安全执行激励轨迹并采集到稳定数据。
2. **可辨识**：观测矩阵在真实工况下有足够秩和可接受条件数。
3. **可验证**：辨识参数能在独立验证轨迹上预测关节力/扭矩。

---

## 2. 对应仓库中可复用的核心资产

- 轨迹优化入口：`Optimize_Trajectory.m`
  - 通过 `fmincon` 优化傅里叶轨迹，使归一化观测矩阵条件数最小。
  - 已包含关节角、推杆行程、速度、加速度等约束。
- 回归矩阵计算：`calc_W_base.m`
  - 由 Symbolic Toolbox 自动生成，适合在线/批处理快速计算。
- 傅里叶运动学还原：`calc_fourier_kinematics.m`
  - 给定傅里叶参数，生成 `q/dq/d2q`。
- 二次可辨识分析：`untitled4.m`
  - 对实用工作空间下的 `W` 做 SVD 与 QR 列筛选。
- 机构运动学映射：`kine_p_2.m`
  - 可用于将末端/几何关系映射到关节与推杆长度，并叠加装配补偿量。

## 3. 实机实施建议流程（推荐按周推进）

### 第 0 周：安全和接口准备

- 在 PLC/控制器设置双层保护：
  - 软件限位（角度、速度、加速度、行程）。
  - 硬件急停和电流阈值。
- 确认采样链路时间戳统一（控制器时钟 vs 上位机时钟）。
- 明确记录字段：
  - `t, q, dq, d2q, tau(or force), motor_current, mode_flag, estop_flag`。

### 第 1 周：离线轨迹先跑通

1. 在仿真/离线先运行 `Optimize_Trajectory.m` 得到 `Optimized_Fourier_Coeffs.mat`。
2. 运行 `untitled4.m`，确认奇异值衰减和最终条件数在可接受范围。
3. 从优化轨迹中导出控制器可下发的时序点（建议 100~500 Hz 插值）。

### 第 2 周：低风险实机试运行

- 以 20%~30% 速度比例执行轨迹，先做空载。
- 首次只做 1~2 个周期，排查：
  - 是否触发限位；
  - 电流是否异常；
  - 采样是否丢包。
- 通过后逐步提升到目标速度并记录完整数据。

### 第 3 周：参数估计与验证

- 构造线性回归：
  - `tau = W_base(q,dq,d2q) * pi_base`
- 估计方法建议：
  1. 加权最小二乘（WLS）作为基线；
  2. 结合岭回归（抑制病态）；
  3. 必要时引入鲁棒损失（Huber）。
- 验证：
  - 使用与训练不同的轨迹做前向预测；
  - 评估 RMSE、MAPE、残差频谱。

## 6. 典型失败模式与应对

- **条件数很高，参数抖动大**：
  - 提升轨迹激励丰富度（调整傅里叶系数初值/谐波数 N）；
  - 增加激励周期与多工况数据拼接；
  - 做参数分组辨识（先质量项，再惯量项）。

- **实机执行越界/报警**：
  - 缩小轨迹幅值并重跑优化；
  - 强化约束（把软限位改得更保守）。

- **估计结果不稳定、跨天漂移**：
  - 纳入摩擦/温漂项；
  - 固定预热流程后再采集；
  - 将温度作为额外回归量。
